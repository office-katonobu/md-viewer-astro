# 🚀 Astro社内ドキュメントシステム開発：フェーズ3継続用コンテキスト

私たちは現在、AstroのContent Collections（`src/content/docs/` 内のMarkdown）を使用した「社内ドキュメントシステム」を開発しています。
フェーズ2までの仕様改善と不具合対応が「完全確定」したため、その前提知識と、これから行うべき「作業予定項目」をここに引き継ぎます。
この前提をすべてインプットした上で、指示に従って開発を継続してください。

---

## 🛠️ 1. 現在のシステム構成と確定コードの前提

### ① ディレクトリとルーティング構造
* `src/content/docs/` 内に多階層の日本語フォルダ・Markdownファイルが配置されています。
* `src/pages/[...slug].astro` が動的ルーティングを担当し、拡張子や `/index` を除外した綺麗なURLを生成します。

### ② 技術的な最重要解決策：「絶対安全な辞書検索（レジストリマッチ）方式」
* Markdown内の相対リンク（`../records/ファイル名.md` など）が日本語URLや階層の深さで破綻するのを防ぐため、サーバーサイドで全ドキュメントの「生のファイル名」と「Astroの正規絶対URL」のペア対応表（`urlRegistry`）を事前に生成しています。
* クライアントサイドのJavaScriptは、ページ内の各リンクとこの辞書を照合し、一致した場合は100%正しい絶対URLパス（％エンコード済）へ `href` を強制上書き補正します。これにより、ルーティング起因の404エラーを根底からシャットアウトしています。

### ③ 実装済みの機能一覧
1. **ドキュメントツリーの並び順最適化:** 同一階層内では、常に「ファイルを先、フォルダを後」の順で並び替えて描画（`BaseLayout.astro` 内）。
2. **サイドバーの初期全閉じ＆URL連動開閉:** 初期表示はフォルダ全閉じ。ページアクセス時に、アドレスバーのURL階層をJavaScriptで解析し、該当する親フォルダ（`<details>`）だけをピンポイントで自動展開。
3. **現在地のメニューハイライト:** 表示中ページのサイドバー項目を、背景色：青（`bg-blue-50 text-blue-600`）に強調表示。
4. **本文内外部リンクの動的抽出と別タブ化:** Markdown本文内の `http(s)://` リンクを正規表現で自動抽出し、サイドバー最下部にリアルタイムでリストアップ。さらに、本文内・サイドバー内を問わず、すべての外部リンクのクリック時に `target="_blank"`（別タブ）を一括注入。
5. **環境変数によるデバッグ制御:** `PUBLIC_ENABLE_DEBUG=true/false`（`.env`）によって、画面最上部に表示される「リアルタイム補正ログ検証パネル」の表示有無を設定可能（裏方の補正ロジックは常に稼働）。`.env.sample` も整備済。

## 📅 2. 次のフェーズにおける作業予定項目（バックログ）

次のステップとして、以下のタスク（機能拡張や本番化への対応）を予定しています。
※次回再開時に、実施したいタスクの番号を指定します。

* **【タスク1】本文（Markdown）のCSSスタイリング強化・レスポンシブ対応**
  Tailwind CSSの `@tailwindcss/typography`（`prose` クラス）を導入していますが、社内文書としてより見やすい見出し、テーブル（表）の罫線、コードブロック、引用（`>`）、警告文（Alertコンポーネント）などの見た目をさらに美しくカスタマイズする。また、スマートフォンやタブレットで見せる場合のサイドバーのレスポンシブ対応（ハンバーガーメニュー化）。
* **【タスク2】全文検索（Search）機能の導入**
  Markdown内のテキストやタイトルから、目的の文書をパッと検索できる軽量な検索窓（PagefindやFlexSearchなど）の設計・組み込み。
* **【タスク3】ドキュメントのメタデータ（フロントマター）の拡張とバリデーション**
  各Markdownの先頭に、`tags`（タグ）、`author`（作成者）、`updatedDate`（更新日）などを追加し、`src/content/config.ts` の `zod` スキーマを厳格に定義して、システム全体のデータ品質を担保する。
* **【タスク4】静的ビルド（Build）と本番サーバーへのデプロイ準備**
  `npm run build` を実行した際の成果物の検証、およびドキュメントルート（サブディレクトリ運用などがある場合）のパス調整、本番環境への安全なデプロイ手順の確立。
