### 4. システムファイルの作成（3つ）

最新のAstro 7仕様に合わせて、必要なフォルダとファイルを配置します。まずはフォルダを作成します。

```bash
mkdir -p src/layouts src/pages src/content/docs

```

#### ① `src/content.config.ts` の作成

**※重要：** `src/content/` の中ではなく、**`src/` 直下**に配置してください。

```typescript
import { defineCollection, z } from 'astro:content';
import { glob } from 'astro/loaders';

const docs = defineCollection({
  loader: glob({ pattern: '**/*.md', base: './src/content/docs' }),
  schema: z.object({
    title: z.string().optional(),
  }),
});
export const collections = { docs };

```

#### ② `src/layouts/BaseLayout.astro` の作成

```astro
---
const { title } = Astro.props;
---
<html lang="ja">
  <head>
    <meta charset="utf-8" />
    <title>{title}</title>
  </head>
  <body style="font-family: sans-serif; padding: 2rem; background: #f8fafc; color: #334155;">
    <main style="max-w: 800px; margin: 0 auto; background: white; padding: 2rem; border-radius: 8px;">
      <slot />
    </main>
  </body>
</html>

```

#### ③ `src/pages/[...slug].astro` の作成

```astro
---
import { getCollection, render } from 'astro:content';
import BaseLayout from '../layouts/BaseLayout.astro';
import path from 'node:path';

export async function getStaticPaths() {
  const docs = await getCollection('docs');
  return docs.map((entry) => {
    const slug = entry.id.replace(/\/index\.md$/, '').replace(/\.md$/, '');
    return {
      params: { slug: slug === 'index' ? undefined : slug },
      props: { entry },
    };
  });
}
const { entry } = Astro.props;
const { Content } = await render(entry);
const pageTitle = entry.data.title || path.basename(entry.id, '.md');
---
<BaseLayout title={pageTitle}>
  <h1>{pageTitle}</h1>
  <Content />
</BaseLayout>

```

今の状態だと、Markdownの見出し（`#`）や本文が、ただの素のテキスト（黒文字・装飾なし）として表示されているかと思います。

次は、プロジェクト定義書に定義した最新の **Tailwind CSS v4** と **Typographyプラグイン（prose）** を導入し、ドキュメントを読みやすく、綺麗に装飾していきます。

以下の手順を順番に進めていきましょう。

### 1. サーバーの停止とパッケージのインストール

一度ターミナルで `Ctrl + C` を押してサーバーを止め、Tailwind v4に必要なパッケージを追加します。以下のコマンドを実行してください。

```bash
npm install @tailwindcss/vite tailwindcss

```

### 2. `package.json` の開発スクリプトの恒久化

今後、常に `--host` モードで安全に起動できるよう、`package.json` の `"dev"` スクリプトを書き換えます。

`package.json` を開き、`"scripts"` の `"dev"` の行を以下のように修正して保存してください。

```json
  "scripts": {
    "dev": "astro dev --host", // 末尾に --host を追記
    "build": "astro build",
    "preview": "astro preview",
    "astro": "astro"
  },

```

### 3. `astro.config.mjs` の書き換え（Tailwind v4の有効化）

Astroの内部エンジン（Vite）に、Tailwind v4のプラグインを組み込みます。中身を以下のように書き換えて保存してください。

```javascript
import { defineConfig } from 'astro/config';
import tailwindcss from '@tailwindcss/vite'; // インポートを追加

export default defineConfig({
  vite: {
    plugins: [tailwindcss()], // Viteのプラグインとして登録
  },
});

```

### 4. グローバルCSSファイルの作成

Tailwind v4の基本スタイルを設定するためのファイルを1つ作ります。

1. `src/styles/` というフォルダを作成します。
2. その中に `global.css` を作成し、以下の**1行だけ**を記述して保存します。

```css
@import "tailwindcss";

```

### 5. レイアウトファイル（`BaseLayout.astro`）の修正

作成したCSSファイルを読み込み、さらにMarkdownが表示される部分にTailwindのドキュメント用クラス（`prose`）を適用します。

`src/layouts/BaseLayout.astro` の中身を、以下に丸ごと書き換えて保存してください。

```astro
---
/* src/layouts/BaseLayout.astro */
import '../styles/global.css'; // Tailwind v4のスタイルをここで読み込む

const { title } = Astro.props;
---
<html lang="ja">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width" />
    <title>{title}</title>
  </head>
  {/* 背景色を薄いグレー(bg-slate-50)に、文字色を少しマイルド(text-slate-800)に */}
  <body class="bg-slate-50 text-slate-800 p-8">
    {/* 中央寄せの白いカード型のコンテナを作成 */}
    <main class="max-w-3xl mx-auto bg-white p-8 rounded-xl shadow-sm">
      {/* prose クラスを付与することで、中のMarkdown（h1, p, ul等）に 
        美しい文字サイズ、余白、箇条書きのスタイルが自動適用されます 
      */}
      <article class="prose prose-slate max-w-none">
        <slot />
      </article>
    </main>
  </body>
</html>

```

## 🛠️ ステップ 1-3：相対リンク自動変換プラグインの適用

現在、Markdown内のリンク（例：`[詳細](./sub/detail.md)`）はそのまま `.md` というファイル名のまま出力されてしまうため、クリックするとWebサイト上では「404 Not Found（ページが見つからない）」になってしまいます。

これを、Astroが解釈できるWeb用のURL構造（拡張子の `.md` を自動で消去する）にビルド時に変換するプラグインを導入します。

最新のAstro 7（Sätteriエンジン）でこのJavaScript製プラグインを安全に動作させるため、プロジェクト定義通りに設定していきます。

### 1. サーバーの停止とパッケージのインストール

一度ターミナルで `Ctrl + C` を押してサーバーを止め、Markdownの解析に必要なパッケージを追加します。以下のコマンドを実行してください。

```bash
npm install @astrojs/markdown-remark unist-util-visit

```

### 2. `astro.config.mjs` の書き換え

インポートに `{ visit }` と `{ unified }` を追加し、プラグインの処理を組み込みます。中身を以下のように丸ごと書き換えて保存してください。

```javascript
import { defineConfig } from 'astro/config';
import tailwindcss from '@tailwindcss/vite';
import { visit } from 'unist-util-visit'; // 名前付きインポート
import { unified } from '@astrojs/markdown-remark'; // Astro 7互換用エンジン

// Markdown内の「.md」への相対リンクを、Web用のURL構造に自動変換するカスタムプラグイン
function remarkFixMarkdownLinks() {
  return (tree) => {
    visit(tree, 'link', (node) => {
      // 外部リンク（httpから始まるもの）を除外し、末尾が .md のリンクを対象にする
      if (!node.url.startsWith('http') && node.url.endsWith('.md')) {
        // /index.md はスラッシュごと消し、それ以外の .md も綺麗に消去する
        node.url = node.url.replace(/\/index\.md$/, '').replace(/\.md$/, '');
      }
    });
  };
}

export default defineConfig({
  vite: {
    plugins: [tailwindcss()],
  },
  markdown: {
    // Astro 7の新エンジン上で従来のプラグインを動作させるための最新の記述方法
    legacy: unified({
      remarkPlugins: [remarkFixMarkdownLinks],
    })
  }
});

```
