# フェーズ2
## 📁 フェーズ2：ステップ 2-1「サイドバー（目次ツリー）の自動生成」

現在の状態だと、他の階層のページに移動するには、Markdown内の相対リンクを辿るしかありません。
これを、よくある技術ドキュメントやWikiのように、**画面の左側にフォルダ階層とファイル名が自動でリストアップされる「サイドバー」** を作成します。

執筆者が新しいMarkdownファイルを配置したら、Astroが自動的にそれを検知してサイドバーのメニューにも自動で追加される仕組みにします。

このステップでは、以下の3つの作業を行います。

1. **メニュー用データの自動生成（Astro側での解析）**
2. **レイアウトの2カラム化（左：サイドバー、右：メイン本文）**
3. **デザインの調整**

準備はよろしいでしょうか？まずは、フォルダ階層を解析してメニュー用のデータを組み立てるために、レイアウトファイルを修正していきましょう！

---

### 🛠️ 1. `src/layouts/BaseLayout.astro` の修正（サイドバーの組み込み）

画面を「左側のメニュー（サイドバー）」と「右側の本文」の2カラム（横並び）に変更し、Astroの機能で全Markdownファイルから自動的に目次ツリーを作ります。

`src/layouts/BaseLayout.astro` の中身を、以下に丸ごと書き換えて保存してください。

```astro
---
/* src/layouts/BaseLayout.astro */
import '../styles/global.css';
import { getCollection } from 'astro:content';
import path from 'node:path';

const { title } = Astro.props;

// 1. 全てのドキュメント（Markdown）を取得
const allDocs = await getCollection('docs');

// 2. サイドバー用にデータを整理（URLとタイトル、階層情報を並べる）
// ※ index.md は除外し、各ページのリンクを生成します
const sidebarItems = allDocs
  .map((doc) => {
    const slug = doc.id.replace(/\/index\.md$/, '').replace(/\.md$/, '');
    const url = slug === 'index' ? '/' : `/${slug}`;
    const displayTitle = doc.data.title || path.basename(doc.id, '.md');
    
    // フォルダ階層の深さを測る（スラッシュの数）
    const depth = doc.id.split('/').length - 1;

    return {
      id: doc.id,
      url,
      title: displayTitle,
      depth,
      isIndex: doc.id.endsWith('index.md')
    };
  })
  // ルートの index.md 以外をサイドバーに表示（お好みで調整可能）
  .filter(item => item.url !== '/');

// 現在のページのURL（アクティブなメニューをハイライトするため）
const currentPath = Astro.url.pathname;
---
<html lang="ja">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width" />
    <title>{title}</title>
  </head>
  <body class="bg-slate-50 text-slate-800">
    
    {/* 画面全体を包むコンテナ（横並びのレイアウトを定義） */}
    <div class="flex min-h-screen max-w-7xl mx-auto">
      
      {/* ─── 左側：サイドバーメニュー ─── */}
      <aside class="w-64 bg-white border-r border-slate-200 p-6 sticky top-0 h-screen overflow-y-auto">
        <div class="mb-6">
          <a href="/" class="text-xl font-bold text-slate-900 no-underline">🏠 ドキュメントTOP</a>
        </div>
        
        <nav class="space-y-1">
          <p class="text-xs font-semibold text-slate-400 uppercase tracking-wider mb-2">ページ一覧</p>
          <ul class="space-y-1 pl-0 list-none">
            {sidebarItems.map((item) => {
              // 現在開いているページかどうかを判定
              const isActive = currentPath === item.url || currentPath === `${item.url}/`;
              
              // 階層（depth）に応じて左側に余白（インデント）を作る
              const paddingLeft = item.depth > 0 ? `${item.depth * 12}px` : '0px';

              return (
                <li style={{ paddingLeft }}>
                  <a
                    href={item.url}
                    class={`block py-1.5 px-3 rounded-lg text-sm font-medium transition-colors no-underline ${
                      isActive
                        ? 'bg-blue-50 text-blue-600 font-semibold' // アクティブ時のデザイン
                        : 'text-slate-600 hover:bg-slate-100 hover:text-slate-900' // 通常時のデザイン
                    }`}
                  >
                    {item.isIndex ? `📁 ${item.title}` : `📄 ${item.title}`}
                  </a>
                </li>
              );
            })}
          </ul>
        </nav>
      </aside>

      {/* ─── 右側：メイン本文 ─── */}
      <main class="flex-1 bg-white p-12 overflow-x-hidden">
        <div class="max-w-3xl mx-auto">
          <article class="prose prose-slate max-w-none">
            <slot />
          </article>
        </div>
      </main>

    </div>

  </body>
</html>

```

## 🏁 動作確認

ファイルを保存したら、ブラウザでサイト（`http://localhost:4321/`）を確認してください。

* [ ] 画面の左側に、手動配置したMarkdownのファイル名（またはFrontmatterのタイトル）が一覧としてズラリと並んでいますか？
* [ ] ディレクトリの中にあるファイル（深い階層のもの）は、自動的に少し右側にずれて（インデントされて）表示されていますか？
* [ ] サイドバーのメニューをクリックしたとき、右側の本文がパッと切り替わり、現在開いているメニューが青くハイライトされますか？

まずは、この「全ファイルをフラットに並べて階層分インデントする」というシンプルな自動サイドバーが動くか確認してみてください。結果を楽しみに待っています！
